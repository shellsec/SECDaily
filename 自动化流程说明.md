# 每日自动分析和总结流程说明

## ✅ 已实现的自动化功能

### 完整流程

每天自动执行以下流程（默认时间：每天10:00）：

1. **收集安全资讯** (`yarb.py` → `job()`)
   - 从配置的RSS源收集当天的安全资讯
   - 过滤和整理文章

2. **生成Markdown文件** (`yarb.py` → `update_today()`)
   - 生成 `today.md` 文件
   - 生成 `archive/年份/日期.md` 文件（如：`archive/2025/2025-11-06.md`）

3. **自动AI总结** (`yarb.py` → `update_today()` → `oneapi.py`)
   - 自动调用 `summarize_security_news()` 函数
   - 对当天的安全资讯内容进行AI分析和总结
   - 自动保存到 `archive/年份/AISummary日期.md` 文件（如：`archive/2025/AISummary2025-11-06.md`）

4. **生成HTML报告** (`yarb.py` → `job()` → `convert_today.py`)
   - 自动调用 `convert_today.py` 生成HTML文件
   - HTML文件自动加载AI总结内容
   - 在页面右侧浮窗显示AI总结

5. **推送通知** (可选)
   - 通过配置的机器人推送结果

## 📋 执行时间

- **默认时间**：每天 10:00
- **可配置**：通过 `--cron` 参数自定义时间，如 `--cron "11:00"`
- **手动执行**：使用 `--force` 参数立即执行

## 📁 生成的文件

每天自动生成以下文件：

1. **today.md** - 当天的安全资讯（根目录）
2. **archive/年份/日期.md** - 归档的安全资讯（如：`archive/2025/2025-11-06.md`）

## 🚀 使用方法

### 生产环境部署

1. **后台运行**（推荐使用系统服务或screen/tmux）：
   ```bash

   screen -S yarb
   python yarb.py
   # 按 Ctrl+A 然后 D 退出screen

   nohup python yarb.py > yarb.log 2>&1 &
   ```


## 🔧 故障排查

### AI总结失败

1. 检查 `ai_config.py` 配置是否正确
2. 检查网络连接
3. 查看日志中的错误信息

### HTML生成失败

1. 检查 `convert_today.py` 是否正常
2. 检查模板文件是否存在
3. 查看日志中的错误信息

### 定时任务不执行

1. 检查系统时间是否正确
2. 检查程序是否在运行
3. 查看日志文件

## 📝 注意事项

1. **AI总结需要时间**：根据内容长度，AI总结可能需要30秒到2分钟
2. **网络要求**：需要能够访问AI API
3. **存储空间**：每天生成的文件会占用一定存储空间
4. **错误处理**：即使AI总结失败，主流程仍会继续执行
